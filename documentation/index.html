WooCommerce Elavon Payment Gateway Plugin

Elavon's Virtual Merchant Payment Gateway

  • Created: 08/24/2012
  • Updated: 02/25/2015
  • By: Jacob M Davidson
  • Email: jacobdavidson79@outlook.com

The WooCommerce Elavon/Virtual Merchant Gateway plugin enables you to take credit card payments directly from your WooCommerce site using a Virtual Merchant account provided by Elavon.

System Requirements

The Elavon/Virtual Merchant payment gateway plugin for woocommerce requires the following setuo:

  1. An installation of wordpress 3.3 or above. Wordpress may be downloaded from http://wordpress.org/.
  2. The latest version of WooCommerce. WooCommerce may be downloaded from http://www.woothemes.com/woocommerce/.
  3. An Elavon/Virtual Merchant account. Go to http://www.elavon.com/ for more information.

To use this plugin, you must have the force secure checkout option enabled within WooCommerce, and a valid SSL certificate (all of which is required prior to Elavon/Virtual Merchant's initial account approval process).

Installation

 

  1. From your Wordpress admin dashboard, select "Plugins"
  2. Select the link "Add New" and choose "Upload"
  3. Select "Choose File"
  4. Browse to the downloaded "woocommerce-gateway-elavon-virtualmerchant.zip" file and select "Open"
  5. Select "Install Now"
  6. Click on the "Activate Plugin" link, and the plugin is installed!
 

Configuration

To configure the plugin, select "WooCommerce" from your admin dashboard and follow these steps:

  1. Click on the "Settings" link
  2. Select the "Payment Gateways" tab
  3. Click on the "VirtualMerchant" link
  4. WARNING: Verify that the currency setting for WooCommerce matches the currency setting for your Virtual Merchant account.
  5. Enter the title you wish to be displayed for this payment option on the Checkout page of your site. The default is "Credit Card/Debit Card".
  6. Check the box to enable VirtualMerchant.
  7. Provide a description to be displayed below the title on the Checkout page.
  8. If you wish to test the plugin, enable the "VirtualMerchant Test" option. A unique testing account from Virtual Merchant is required for the testing option to work. Contact Virtual Merchant at 1-800-377-3962 to request a unique testing account.
  9. Provide the Merchant ID, User ID and pin provided by Elavon/Virtual Merchant for your account.
  10. If you wish to accept business credit cards / purchasing cards, select the box to enable business cards.
  11. Choose which credit cards you would like to enable on the checkout page. You can choose from the four major cards accepted by Elavon/Virtual Merchant, Visa, Discover, MasterCard and American Express.
  12. Select "Enable CSC Authentication" if you would like to test the card security code for validation as a condition of sale approval.
  • To enable the credit card security number authentication, you must configure your virtual merchant account in addition to enabling the CSC feature of this plugin.  To configure your virtual merchant account, login to the virtual merchant terminal at myvirtualmerchant.com and create a Post-Processing Rule under "Business Rules" that declines the card when the
    CVV2 response does not equal M.  Contact Elavon at 1-800-377-3962 for further assistance.
  • The CSC Authentication will not work with a testing account. This feature only works with a live Elavon/Virtual Merchant account.

Configuration for the plugin is now complete. Please note, if you're using a demo account, the credit card will randomly be denied for testing purposes.
